## Philosophy

**Deterministic code templating is brittle.** Airflow and Prefect are architecturally dissimilar—edge cases are inevitable.

Instead, we provide **rich structured analysis payloads** that enable LLMs to generate complete, functional Prefect flows using their knowledge of Python and Prefect patterns.

## Workflow

```
1. analyze(dag)     → Comprehensive DAG structure & patterns
2. get_context()    → Relevant Prefect documentation
3. LLM generates    → Complete Prefect flow code
4. validate()       → Verify structural correctness
```

**We analyze. The LLM generates. We validate.**

## What the tools provide

| Tool | Returns |
|------|---------|
| `analyze` | Structure, patterns, config, complexity, migration notes |
| `get_context` | Prefect docs, operator mappings, deployment templates |
| `operator_mapping` | Prefect equivalent for specific operator |
| `connection_mapping` | Prefect block type for Airflow connection |
| `validate` | Structural comparison and coverage report |
